The alphabet A through M is here: this test case is specially constructed so that when the view is sized to 800x600 (the default for DumpRenderTree) the text wraps and not all of the text in the SVG is drawn correctly. You should see all of A-L (and half of M) above. The code was incorrectly calculating the PaintInfo::rect (damage rect) when passing it through RenderBlock and render block was (correctly) clipping out some of the lineboxes during paint.
